# GETsvg 地图包修正数据示例

## 目标

这份文档给后端作为正确数据示例使用。

原则：

- `SvgUrl` 指向实际可访问的 SVG 资源。
- `CoordinateTransform.SvgBounds` 使用 CAD 主体图层范围，不使用整张 SVG viewBox。
- 设备 `X/Y` 使用 `CadX/CadY` 按正确 transform 重新计算。
- 区域 `SvgPoints` 使用正确 transform 重新计算。
- 不返回 `总图...最大点` 派生出来的假 GW，或明确标记给前端过滤。

## 修正后的 SvgUrl 示例

本地测试路径：

```json
"SvgUrl": "/maps/be5f3560-c683-4073-ace3-9e0e70cd5ff3/05a55c49813d46ec81049767ba1001a7.svg"
```

生产环境应由后端资源服务提供等价可访问 URL，不建议前端写死本地 `dist/maps/...` 文件路径。

## 修正后的 CoordinateTransform

```json
{
  "Type": "affine",
  "A": 0.05037299993227306,
  "B": 0,
  "C": 150.49894467109275,
  "D": 0,
  "E": -0.05037149421480847,
  "F": 645.5128182076019,
  "CadBounds": {
    "MinX": 2471.5831,
    "MinY": 1670.0481,
    "MaxX": 7623.1524,
    "MaxY": 11231.4083,
    "Width": 5151.5693,
    "Height": 9561.3602
  },
  "SvgBounds": {
    "MinX": 275,
    "MinY": 79.77,
    "MaxX": 534.5,
    "MaxY": 561.39,
    "Width": 259.5,
    "Height": 481.62
  },
  "Source": "SVG_LAYER_BOUNDS"
}
```

计算公式：

```text
svg.x = cad.x * A + cad.y * B + C
svg.y = cad.x * D + cad.y * E + F
```
