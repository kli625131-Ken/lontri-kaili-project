# 数字底图进展存档

存档日期：2026-07-07

## 一、数字底图概述

数字底图当前基于现有 `DeviceVisualization` 页面实现，不新建重复页面。页面以本地 SVG 底图和本地 JSON 数据为基础，完成厂房/楼层地图展示、区域覆盖、CU/GW 设备点位叠加、缩放平移、区域聚焦、图层筛选、缩略图联动和抽屉查看控制等功能。

当前地图加载链路为：

```text
楼层配置
→ SVG 底图
→ meta 坐标转换配置
→ data 区域与设备数据
→ CAD 坐标转换为 SVG 坐标
→ SVG 同坐标系渲染底图、区域、设备、遮罩和标签
```

核心文件：

- 页面：`src/views/DeviceVisualization.vue`
- 楼层配置：`src/config/deviceVisualizationMaps.js`
- 地图底图：`public/maps/floor1.svg`、`public/maps/lvdi_1F.svg`
- 地图数据：`public/maps/floor1.data.json`、`public/maps/floor1.meta.json`、`public/maps/lvdi_1F.data.json`、`public/maps/digital-map-test.data.json`
- 设备图标：`src/assets/images/devices/CU.png`、`src/assets/images/devices/GW.png`
- 坐标/几何工具：`src/utils/deviceMapGeometry.js`
- 本地草稿服务：`src/services/deviceMapLocalService.js`
- 配置调试组件：`src/components/deviceMap/MapModeSwitch.vue`、`MapEditToolbar.vue`、`MapConfigDrawer.vue`

## 二、当前进展

### 1. 真实地图数据加载

- 已停用随机设备、随机区域和 fallback 坐标作为正式展示数据。
- 开利 101 车间已使用 `floor1.svg`、`floor1.meta.json`、`floor1.data.json` 加载真实区域和设备。
- 当前验证结果：CU 设备 404 个、GW 设备 6 个、区域 6 个。
- 区域、设备和 SVG 底图位于同一个根 SVG 坐标系，缩放、重置、区域聚焦后保持同步。

### 2. 坐标转换

- CAD/SVG 坐标转换参数集中保存在 `floor1.meta.json`。
- 前端统一通过 `cadToSvg(point)` 和 `svgToCad(point)` 完成坐标换算。
- 设备点位只执行一次 CAD 到 SVG 转换，避免二次缩放导致底图和点位错位。
- 区域编辑、设备移动、新增设备保存草稿时复用 `svgToCad()` 生成 CAD 坐标。

### 3. 设备与区域展示

- CU/GW 均使用 `src/assets/images/devices/` 下真实图片图标，通过 SVG `<image>` 渲染。
- 默认完整地图状态下不显示 404 个设备编号，避免地图拥挤。
- 进入区域聚焦后，仅显示当前区域内设备编号，格式使用 `GWNO  ZIGBEENO`。
- 区域名称和数量已调整为更适合当前地图的显示方式，减少与设备图标遮挡。
- 已移除设备持续晃动、呼吸、闪烁类动画，设备默认保持静止。

### 4. 地图交互

- 保留滚轮缩放、重置视图、全图查看。
- 支持鼠标中键按住拖动平移地图。
- 左上角保留图层筛选，并增加地图缩略图。
- 缩略图显示完整地图、区域轮廓、简化设备点和当前视口范围。
- 点击区域后支持自动聚焦、居中放大、区域外暗化、当前区域高亮。
- 点击空白区域、按 Esc、点击重置视图可退出区域聚焦。
- 点击 CU/GW 仍打开原设备抽屉；点击区域仍打开原区域抽屉。

### 5. 空间层级导航

当前空间层级已整理为：

```text
绿地办公室项目
└─ 上海闵行
   ├─ 绿地科技广场
   │  └─ 10楼
   │     ├─ 一层
   │     ├─ 一层测试
   │     └─ 二层
   └─ 开利101车间
      └─ 一楼车间
```

- 层级菜单支持展开和收缩。
- 点击末级楼层节点后，前端切换对应 SVG、meta、data、区域、设备和缩略图。
- 楼层切换前会清除聚焦区域、选中设备、抽屉、绘制草稿、缩放和平移状态。

### 6. 样式优化

- 数字地图容器、SVG 底图背景、CAD 线稿、区域色块已按当前系统深色风格做过一版配色优化。
- 底图线稿颜色调整为更清晰的深灰蓝系，避免过深或压过区域与设备。
- 地图底部中间已集中放置缩放、重置视图、全图控制。
- 左下角原提示类控件已减少干扰，地图主体信息层级更清晰。

### 7. 双模式改造

已完成前端第一版双模式框架：

- 默认模式：用于查看地图、区域聚焦、图层筛选、缩放平移、查看区域/CU/GW 抽屉和执行现有前端控制逻辑。
- 配置调试模式：进入前弹确认框，进入后显示地图编辑工具栏。

配置调试模式当前包含：

- 选择
- 新建矩形区域
- 新建圆形区域
- 新建多边形区域
- 编辑区域
- 删除区域
- 新增设备
- 移动设备
- 删除设备
- 区域组配置
- 场景配置
- CU 参数
- 测试校验

### 8. 前端本地草稿

- 已新增前端本地运行态数据，不直接修改原始地图 JSON。
- 草稿数据包含：`regions`、`devices`、`areaGroups`、`scenes`、`cuParams`、`draftState`。
- 草稿保存到 `localStorage`，不调用后端接口。
- 当前不做 JSON 导入、导出。

### 9. 区域、设备和区域组编辑

- 配置调试模式下支持矩形、圆形、多边形区域创建。
- 区域编辑后会重新计算区域内设备。
- 支持新增、移动、删除设备。
- 移动设备后使用 `svgToCad()` 更新 CAD 坐标，并重新计算所属区域。
- 删除设备后同步更新区域设备数量和本地区域组成员。
- 网关控制和区域组控制已按两套独立逻辑处理。
- 区域组只允许同一网关下 CU 组成；检测到跨网关设备时阻止保存并提示。
