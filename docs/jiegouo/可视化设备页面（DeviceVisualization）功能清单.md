# 可视化设备页面（DeviceVisualization）功能清单

## 1. 结论

当前“可视化设备”页面是一个基于 Vue 3 的楼层数字地图页面，已具备真实楼层底图加载、CAD 坐标数据叠加、CU/GW 设备点位展示、区域聚焦、设备/区域控制抽屉、临时区域绘制和开发调试面板等前端功能。

当前控制类操作仍停留在前端本地状态修改，没有接入后端接口、真实设备下发、状态回写和持久化保存。

## 2. 页面入口

- 路由路径：`/device-visualization`
- 页面标题：`可视化设备`
- 页面组件：`src/views/DeviceVisualization.vue`
- 依赖组件：`src/components/DataCard.vue`
- 底图资源：`public/maps/floor1.svg`
- 地图元数据：`public/maps/floor1.meta.json`
- 地图业务数据：`public/maps/floor1.data.json`

## 3. 页面布局功能

### 3.1 空间层级导航

- 展示空间层级树：
  - 园区
  - 包装中心
  - 一层
  - AR包装厂房
  - 灯检走廊
  - 输送缓冲区
- 默认高亮 `AR包装厂房`。
- 展示统计摘要：
  - CU 数量
  - GW 数量
  - 区域数量
- 统计数据来自页面当前加载后的响应式数据。
- 当前空间树是静态模板，不支持点击切换楼层或区域联动。

### 3.2 数字地图区域

- 展示楼层 SVG 底图。
- 在同一 SVG 坐标系内叠加：
  - 区域 polygon 图层
  - GW 网关图标
  - CU 设备图标
  - 聚焦遮罩
  - 绘制草稿图层
  - Debug 叠加图层
- 展示地图图例：
  - CU设备
  - GW设备
  - 区域图层
- 展示底部操作提示：
  - 普通模式：滚轮缩放 1x - 5x，点击图层对象打开控制抽屉
  - 绘制模式：左键连续落点，右键结束并自动闭合区域
- 地图数据加载失败时展示错误提示。

## 4. 地图工具栏功能

### 4.1 缩放倍率展示

- 实时显示当前缩放倍率，例如 `缩放 1.0x`。
- 缩放倍率由原始 viewBox 宽度与当前 viewBox 宽度计算。

### 4.2 区域绘制开关

- 按钮文案：
  - 未绘制：`区域绘制`
  - 绘制中：`绘制中`
- 点击后进入或退出区域绘制模式。
- 进入绘制模式时会关闭当前抽屉、清空聚焦区域和提示信息。

### 4.3 重置视图

- 点击 `重置视图` 后恢复到原始 viewBox。
- 会停止正在进行的区域聚焦动画。

### 4.4 清空草稿

- 点击 `清空草稿` 后清空当前绘制点位和鼠标预览点。
- 当没有绘制点位时按钮禁用。

### 4.5 设备图层筛选

- 支持三个筛选项：
  - `全选`：显示 CU 和 GW
  - `CU`：只显示 CU
  - `网关`：只显示 GW
- 筛选只影响设备图标，不影响区域图层。

### 4.6 Debug 开关

- 仅开发环境显示。
- 开启后显示 SVG Debug 面板和调试叠加层。
- 当前页面主流程使用 `floor1.data.json` 真实数据，Debug 统计展示的是当前叠加数据映射数量。

## 5. 数据加载与坐标转换功能

### 5.1 楼层资源加载

页面挂载后执行楼层加载流程：

1. 请求 `/maps/floor1.svg`。
2. 请求 `/maps/floor1.meta.json`。
3. 根据 `meta.dataUrl` 请求 `/maps/floor1.data.json`。
4. 使用 `meta.viewBox` 或 SVG 自身 viewBox 初始化地图视图。
5. 标准化 SVG 底图颜色和填充样式。
6. 将 SVG 内部内容注入到地图底图图层。
7. 将业务数据转换成区域、CU、GW 叠加层。

### 5.2 真实数据来源

- `floor1.data.json` 由 `scripts/build-floor1-map-data.js` 从 `docs/开利101车间-灯控设备坐标值260226.xls` 生成。
- 当前数据量：
  - 区域：6 个
  - 设备总数：410 个
  - CU：404 个
  - GW：6 个
- 源表中只有 CU 坐标；GW 图标按区域中心派生。

### 5.3 坐标转换

- 支持 CAD 坐标转换为 SVG 坐标。
- 支持 SVG 坐标反算为 CAD 坐标。
- 当前使用 `floor1.meta.json` 中的 `coordinateTransform` 仿射变换参数。
- 区域顶点和设备点位都会转换到 SVG 坐标系后渲染。

### 5.4 设备初始化字段

从 `floor1.data.json` 生成设备对象时，页面会补齐：

- `id`
- `name`
- `shortName`
- `type`
- `regionId`
- `sourceRow`
- `sourceFields`
- `cadX`
- `cadY`
- `x`
- `y`
- `online`
- `power`
- `mode`
- `brightness`
- `bgBrightness`
- `bgTime`
- `offTime`
- `manualTime`
- `firmware`
- `updatedAt`

默认规则：

- CU 默认开灯、亮度 100%、背景亮度 40%、自动模式。
- GW 默认关闭。
- 在线状态默认 `true`。
- 固件和实时状态显示为 `未接入`。

## 6. 地图交互功能

### 6.1 滚轮缩放

- 鼠标滚轮向上放大，向下缩小。
- 缩放范围：1x 到 5x。
- 以鼠标所在地图坐标为缩放中心。
- 通过修改 SVG viewBox 实现缩放。
- 自动限制视图不超出原始地图范围。

### 6.2 点击空白重置

- 点击地图空白区域、底图背景、聚焦遮罩或底图图层时，重置地图视图。

### 6.3 区域点击聚焦

- 点击区域 polygon 后：
  - 根据区域边界计算目标 viewBox
  - 带 10% padding 聚焦到区域
  - 执行 240ms 动画
  - 打开区域控制抽屉
  - 显示聚焦提示
- 再次点击当前已聚焦区域会取消聚焦。

### 6.4 聚焦遮罩

- 聚焦区域时，非聚焦区域会变暗。
- 非聚焦区域内的设备图标会降低透明度。
- 聚焦区域外增加遮罩，突出当前区域。

### 6.5 设备点击

- 点击 GW 图标打开网关控制抽屉。
- 点击 CU 图标打开 CU 控制抽屉。
- 当前选中设备显示高亮描边。
- 离线样式已预留，但当前数据默认全部在线。

## 7. 区域绘制功能

### 7.1 绘制流程

- 点击 `区域绘制` 进入绘制模式。
- 左键连续添加区域顶点。
- 鼠标移动时显示预览线和预览 polygon。
- 第一个点有起点提示圆。
- 右键结束绘制并自动闭合区域。
- 点击 `清空草稿` 可清除当前点位。

### 7.2 生成规则

- 至少需要 3 个点才能生成区域。
- 少于 3 个点时提示 `区域绘制至少需要 3 个点`，并退出绘制模式。
- 成功生成后：
  - 区域名称为 `新区域-${regions.length + 1}`
  - 自动计算区域内包含的 CU 设备
  - 自动生成场景配方
  - 追加到当前区域列表
  - 打开区域控制抽屉
  - 显示生成成功提示

### 7.3 当前限制

- 绘制区域只保存在前端内存中。
- 刷新页面后丢失。
- 不支持保存、编辑、删除、重命名。
- 不支持与后端区域配置同步。

## 8. 控制抽屉功能

### 8.1 抽屉通用功能

- 右侧滑入式控制抽屉。
- 抽屉类型：
  - CU
  - GW
  - 区域
- 抽屉头部展示：
  - 类型图标
  - 标题
  - 副标题
  - 在线/离线状态
  - 关闭按钮
- 关闭按钮可关闭当前抽屉。

### 8.2 CU 控制抽屉

CU 抽屉标题：`控制面板-CU`

已展示/可操作功能：

- 运行控制：
  - 开启
  - 关闭
- 控制模式：
  - 手动
  - 自动感应
- 亮度设置：
  - 开机亮度，范围 0 - 100%
  - 背景亮度，范围 0 - 100%
- 无人感配置策略：
  - 进入背景亮度时间，单位秒
  - 进入关灯时间，单位秒
  - 手动模式持续时间，单位秒
- 设备状态：
  - 固件版本
  - 最新查询
- 修改参数配置按钮。

当前行为：

- 开关、模式、亮度和时间参数直接修改前端响应式对象。
- `修改参数配置` 只显示本地提示。
- 没有接口提交、loading、失败回滚和权限校验。

### 8.3 GW 控制抽屉

GW 抽屉标题：`控制面板-网关`

已展示/可操作功能：

- 场景控制：
  - 开启
  - 关闭
- 设备状态：
  - 固件版本
  - 最新查询

当前行为：

- 开关状态直接修改前端响应式对象。
- 没有真实网关指令下发。

### 8.4 区域控制抽屉

区域抽屉标题：`控制面板-区域`

已展示/可操作功能：

- 场景控制：
  - 开启
  - 关闭
  - 会议模式
  - 讨论模式
- 场景配方参数只读预览：
  - 设备名称
  - 亮度
  - 色温
- 场景 tab 切换。
- 区域内没有 CU 设备时显示空状态：`当前区域内暂无关联 CU 设备`。

当前行为：

- 切换场景只改变区域对象的 `sceneMode`。
- 场景配方只读展示，不会下发到设备。

## 9. 区域场景配方功能

页面会按区域内 CU 设备生成 4 套配方：

| 场景 | 亮度规则 | 色温规则 |
| --- | --- | --- |
| 开启 | 全部 100% | 4000K |
| 关闭 | 全部 0% | -- |
| 会议模式 | 第 1 台 0%，其余 100% | 4000K |
| 讨论模式 | 第 1 台 60%，其余 80% | 3500K |

区域内设备通过点在多边形内算法计算。

## 10. Debug 功能

### 10.1 Debug 面板

开发环境打开 Debug 后显示：

- Yellow
- Regions
- Cyan
- CU
- GW
- Unmapped candidates
- Warnings

### 10.2 当前 Debug 数据

当前页面加载真实 `floor1.data.json`，不是旧的 SVG 颜色识别流程，因此：

- `Regions` 显示当前加载区域数量。
- `CU` 显示当前映射 CU 数量。
- `GW` 显示当前映射 GW 数量。
- `Yellow` 和 `Cyan` 统计为 0。
- 区域候选框和未映射设备候选点通常为空。

### 10.3 保留的旧识别工具

项目中仍保留：

- `src/utils/floorSvgRecognition.js`
- `src/config/floorRecognition/floor1.overrides.json`

这些文件用于旧的 SVG 颜色识别方案，但当前 `DeviceVisualization.vue` 主流程没有调用。

## 11. 错误处理功能

- 如果 SVG、meta 或 data 任一资源请求失败：
  - 清空地图元数据
  - 清空识别调试信息
  - 展示错误提示
  - 清空 CU、GW 和区域图层
- 当前错误提示文案：`floor1 地图真实数据加载失败，请检查 floor1.svg、floor1.meta.json 和 floor1.data.json。`

## 12. 响应式与显示适配

- 页面默认左右两栏：
  - 左侧空间导航
  - 右侧数字地图
- 低于 1080px 时切换为上下布局。
- 低于 768px 时：
  - 工具栏左对齐换行
  - 图例和状态条改为左右铺满
  - 抽屉改为底部面板样式
  - CU 亮度设置从两列改为一列

## 13. 已实现功能汇总

- 可进入可视化设备页面。
- 可加载真实楼层 SVG 底图。
- 可加载楼层 meta 和业务 data。
- 可展示 6 个区域、404 个 CU、6 个 GW。
- 可进行 CAD 坐标到 SVG 坐标转换。
- 可渲染区域图层。
- 可渲染 CU/GW 设备图标。
- 可展示地图图例和操作状态提示。
- 可滚轮缩放地图。
- 可点击空白重置视图。
- 可点击区域聚焦并打开区域抽屉。
- 可点击 CU/GW 打开对应控制抽屉。
- 可筛选 CU/GW 设备图层。
- 可绘制临时自定义区域。
- 可自动计算区域内 CU 设备。
- 可预览区域场景配方。
- 可本地切换 CU 开关、模式、亮度和无人感时间。
- 可本地切换 GW 开关。
- 可本地切换区域场景模式。
- 开发环境可查看 Debug 面板。
- 地图资源加载失败时有错误提示。

## 14. 当前未实现或未闭环功能

- 空间层级树不可点击。
- 空间层级树未与地图区域联动。
- 不支持多楼层切换。
- 不支持地图拖拽平移。
- 不支持搜索设备或区域。
- 不支持设备标签常显。
- 不支持设备在线状态实时刷新。
- 不支持后端设备控制接口。
- 不支持控制操作确认。
- 不支持控制操作 loading、失败、超时和回滚。
- 不支持操作审计日志。
- 不支持区域绘制结果持久化。
- 不支持区域编辑、删除、重命名。
- 不支持区域场景配置保存。
- 不支持权限控制。
- 不支持 WebSocket、MQTT 或轮询实时状态。

## 15. 建议后续补齐优先级

1. 接入楼层、区域、设备列表接口，替换静态空间树。
2. 接入设备实时状态接口或推送通道。
3. 给 CU/GW/区域控制补充确认、loading、失败回滚和后端下发。
4. 给区域绘制补充保存、编辑、删除、重命名能力。
5. 增加设备搜索、区域搜索和地图拖拽平移。
6. 清理或标注旧 SVG 颜色识别方案，避免维护时误判当前数据来源。
